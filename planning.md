### Tool 1: search_listings

**What it does:**
Searches the mock listings dataset and returns matching items based on user criteria.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): Keywords to match against the listing's title, description, category, or style_tags.
- `size` (str): The specific size string to filter by.
- `max_price` (float): The maximum price limit for the item.

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
A list of dictionaries representing the matching listings. Each dictionary will contain all the standard fields from "listings.json".
**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
If no listings match the filters, the tool returns an empty list rather than throwing an exception. The planning loop will see this empty list and immediately return a string telling the user: "I couldn't find any items matching that description. Want to try loosening the price or removing the size filter?"
---

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Given a newly found secondhand item and the user's current wardrobe, it uses an LLM to suggest one or more complete outfit combinations.
**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): The specific item dictionary selected from the "search_listings" results.
- `wardrobe` (dict): The user's current wardrobe data structure.

**What it returns:**
<!-- Describe the return value -->
A natural language string describing how to style the new item with the existing wardrobe pieces.
**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
The tool must be able to handle an empty or minimal wardrobe. If "wardrobe['items']" is completely empty, the tool must not crash or return an empty string. Instead, it catches the empty state and returns a useful string of general styling advice for the item (e.g., "I see your wardrobe is empty right now! Generally, this vintage graphic tee would look great paired with some baggy, wide-leg jeans and a pair of platform boots.")
---

### Tool 3: create_fit_card

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Generates a short, shareable description of a complete outfit - the kind of thing someone would use as an Instagram caption. It uses the LLM to produce a creative, natural-sounding caption that is different each time it is run.
**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `outfit` (...): The outfit suggesting generated by the "suggest_outfit" tool.

- 'new_item' (dict): The dictionary containing the details of the selected item from the "search_listings" tool.

**What it returns:**
<!-- Describe the return value -->
A short, catchy caption describing the full look.
**What happens if it fails or returns nothing:**
<!-- What should the agent do if the outfit data is incomplete? -->
The tool must guard against receiving an empty or invalid string for the "outfit" parameter. If the outfit string is empty, the tool must not crash or throw an exception. Instead, it returns a descriptive error message string.
